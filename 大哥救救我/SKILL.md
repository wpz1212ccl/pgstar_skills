# 大哥救救我

半自动求助skill：AI生成文档 → 你去智谱提问 → AI读取结果并修改代码。

## 工作流程

### 第一步：生成求助文档

当用户说"大哥救救我"时：

1. 分析当前对话历史，提取：
   - 错误信息和堆栈
   - 相关代码片段
   - 已尝试的方案
   - 项目背景（如果对话中有）

2. 生成求助文档，格式如下：

```markdown
# 求助文档：[问题简述]

## 一、项目背景
[项目简介、技术架构]

## 二、当前实现方式
[核心代码、配置]

## 三、遇到的问题
### 3.1 问题描述
[具体问题]

### 3.2 错误信息
[完整错误日志]

### 3.3 已尝试方案
[已尝试的方法和结果]

## 四、需要帮助的内容
1. [具体问题1]
2. [具体问题2]

## 五、相关文件
[文件路径和说明]

## 六、复现步骤
[如何复现问题]
```

3. 保存到 `求助文档.md`

4. 提示用户：
```
✅ 求助文档已生成：求助文档.md

请按以下步骤操作：
1. 打开文件：求助文档.md
2. 复制全部内容
3. 粘贴到智谱GLM5.2对话框
4. 得到回复后，复制回复内容
5. 粘贴到文件：求助结果.md
6. 回复我："下一步"
```

### 第二步：等待用户操作

用户回复"下一步"时，检查 `求助结果.md` 是否存在：
- 存在 → 进入第三步
- 不存在 → 提示用户先完成操作

### 第三步：读取结果并修改代码

1. 读取 `求助结果.md` 内容

2. 分析智谱的解决方案：
   - 提取关键修改建议
   - 识别需要修改的文件
   - 理解修改逻辑

3. 显示修改方案给用户确认：
```
📋 智谱GLM5.2的解决方案：

## 问题分析
[问题根因]

## 修改方案
1. 修改文件 [文件名]
   - 修改内容：[具体修改]
   
2. 修改文件 [文件名]
   - 修改内容：[具体修改]

是否按此方案修改代码？
```

4. 用户确认后，执行代码修改

5. 修改完成后提示：
```
✅ 代码修改完成

建议你：
1. 测试修改是否解决问题
2. 如果还有问题，可以再次说"大哥救救我"
```

## 提示词模板

生成求助文档时，参考以下提示词格式：

```
你好，我遇到了一个代码问题需要帮助。

## 问题背景
[背景描述]

## 具体问题
[问题描述]

## 已尝试方案
[尝试过的方法]

## 需要帮助
1. [帮助点1]
2. [帮助点2]

如果需要我给你提供什么其他的文件让你更好理解我的项目的话，请告诉我。

请直接给出解决方案，不要输出废话。
```

## 错误处理

- `求助结果.md` 不存在 → 提示用户先完成智谱提问
- 结果内容为空 → 提示用户重新复制
- 无法理解解决方案 → 显示原始内容让用户判断
