# creating-obsidian-vaults

将外部内容（网页/想法/文章）整理进 Obsidian 知识库，保持结构整洁、图文并茂、链接完整。

---

## 触发方式

**关键词触发** — 用户消息包含以下任一关键词时自动启用：
- `记一下`
- `存到知识库`
- `放Obsidian`
- `记录到 Obsidian`
- `帮我存`
- `保存到`

**主动询问** — 无关键词时，如果用户分享有价值的内容（网页链接、技术文章、好的想法等），主动询问：「要不要把这个记到 Obsidian 知识库？」

---

## 知识库路径

```
D:\wpz的知识库\wpz_vault\
```

---

## 工作流程

### 收到网页链接时

1. **抓取内容** — 使用 Playwright 或 webfetch 获取网页完整内容
2. **下载图片** — 所有图片下载到 `assets/<笔记名>/`，Markdown 引用本地路径
3. **分析分类** — 根据内容主题归入最合适的目录
4. **检测链接** — 扫描已有笔记，自动添加 `[[双向链接]]`
5. **撰写笔记** — 按规则生成结构化 Markdown
6. **更新索引** — 更新分类 MOC 和根目录总索引

### 收到想法/笔记时

1. 归入「心里话」或用户指定目录
2. 按同样规则创建笔记（不含来源 URL）

## 分类规则

### 现有目录结构

```
wpz_vault/
├── 求职/                  # 求职面试相关
├── 手机端的输入/           # 手机端捕捉的灵感
│   ├── 灵感捕捉/
│   └── 我的一些计划规划/
├── 我的github 2FA认证/     # 技术配置笔记
├── 心里话/                 # 个人随想、想法（默认归入此类）
├── 学习新东西/             # 新技术/工具学习
├── 学习ai/                 # AI 相关学习
│   ├── ai做游戏/
│   ├── claude-notification/
│   ├── github项目/
│   └── vibe coding/
├── assets/                 # 图片等附件资源
└── how to use obsidian.md
```

### 归类原则

- 根据内容**主题**匹配现有目录名
- 匹配度高的优先（子目录 > 父目录）
- 无匹配时**新建目录**（用简短准确的中文名）
- 无法判断时**问用户**

## 文件命名

```
YYYY-MM-DD-标题.md
```

- 日期使用内容发布日期（网页）或当前日期（想法）
- 标题简短准确，保留中文
- 特殊字符替换为空格

---

## Frontmatter（YAML 头部）

```yaml
---
title: 文章标题
date: 2026-07-17
source: https://原始网页链接
tags:
  - 标签1
  - 标签2
category: 分类名
summary: 一句话摘要
---
```

- `source` — 网页链接（想法类笔记省略）
- `tags` — 从内容自动提取 2-5 个标签
- `category` — 对应目录名

---

## 图片处理

1. **下载**网页中所有图片到 `assets/<笔记名>/`
2. **重命名**为 `图片-1.png`、`图片-2.jpg` 等
3. **引用本地路径**：`![描述](../../assets/笔记名/图片-1.png)`（`../../` 从笔记位置上两级到 vault 根目录）
4. 放在原文对应位置，还原排版

---

## 链接与索引

### 笔记内双向链接

**核心规则：绝对不能创建指向不存在笔记的链接！**

添加 `[[双向链接]]` 前必须执行以下验证流程：

1. **用 Glob 扫描** — 搜索 `D:\wpz的知识库\wpz_vault\**\*.md` 获取所有已有笔记文件列表
2. **用 Grep 按内容匹配** — 在已有笔记中搜索与当前内容相关的关键词（标题、标签、主题）
3. **只链接确认存在的笔记** — 每个 `[[链接]]` 必须对应一个真实存在的 `.md` 文件
4. **按相关度排序** — 优先链接内容最相关的 2-3 个笔记，不要凑数量

**禁止行为**：
- ❌ 凭猜测创建链接（如 `[[2026-07-18-某某项目]]` 但文件不存在）
- ❌ 先写笔记再"补"链接（应该先搜后写）
- ❌ 创建新笔记后链接到它（新笔记没有内容关联，链接无意义）

**搜索策略**：
- 按 **title** 匹配：搜索笔记 frontmatter 中的 `title` 字段
- 按 **tags** 匹配：搜索笔记 frontmatter 中的 `tags` 字段
- 按 **内容关键词** 匹配：Grep 搜索笔记正文中的相关术语
- 按 **同目录** 关联：同一分类下的笔记往往相关

### 分类 MOC 索引

每个分类目录下创建 `README.md`，格式：

```markdown
# 分类名

## 简介
该分类的简要说明

## 笔记列表
- [[2026-07-17-笔记1]] — 一句话简介
- [[2026-07-16-笔记2]] — 一句话简介
```

### 根目录总索引

`wpz_vault/README.md` 汇总所有分类和最新笔记。

## 排版风格

- **美观但不花哨** — 适当使用分隔线 `---`、emoji 点缀
- **还原排版** — 全页搬运时尽量保持原文结构
- **代码原样保留** — 语言标注、格式不变
- **视频/嵌入** — 保留原始链接，不嵌入
- **无固定模板** — 根据内容灵活排版
- **Callout/引用** — 适当使用 Obsidian 的 `>` 引用块

---

## 重复内容处理

检测到已有相同来源的笔记时，**问用户**：
1. 覆盖更新？
2. 新建版本？
3. 跳过？

---

## 语言

保留原文语言。网页是什么语言，笔记就用什么语言。

## 整理知识库

手动触发整理功能，让知识库保持井井有条。

### 触发方式

用户说以下关键词时自动启用：
- `整理一下知识库`
- `帮我整理 Obsidian`
- `清理知识库`
- `整理一下`

### 扫描策略

1. **先读索引** — 读取 `wpz_vault/README.md` 了解全貌，不一开始就扫描所有文件
2. **读取最近变更** — 检查最近修改过的笔记（按文件修改时间）
3. **可指定目录** — 用户说「整理一下学习ai」则只整理该目录

### 清理垃圾

| 类型 | 处理方式 |
|------|---------|
| 空文件夹 | 目录内无 `.md` 文件且无子目录 → 删除 |
| 无意义草稿 | 标题为「未命名」「无标题」「Untitled」等 → 列出清单问用户 |
| 重复笔记 | 同一来源 URL 或内容高度相似 → 列出清单问用户是否合并 |

### 内容整理

- **补充 Frontmatter** — 缺少 title/date/tags/category/summary 的笔记自动补上
- **添加标签** — 根据内容自动提取标签补到已有笔记的 tags 中
- **建立链接** — 扫描所有笔记，内容相关但未链接的补上 `[[双向链接]]`

### 移动文件

发现笔记归类不合理时，问用户确认后再移动。

同步更新：
- 移动后自动更新 `README.md` 索引中的 `[[链接]]`
- 旧位置留下重定向笔记（可选）

### 文件名规范

对于不符合 `YYYY-MM-DD-标题.md` 命名规范的文件，问用户是否统一改名。

### 索引同步

整理后自动更新所有受影响的索引：
- 分类目录 `README.md`
- 根目录 `README.md`

### 输出报告

整理完成后输出变更报告，列出：
- 删除了哪些空文件夹/草稿
- 移动了哪些文件（从哪移到哪）
- 补充了哪些笔记的 Frontmatter/标签
- 建立了哪些新的笔记链接
