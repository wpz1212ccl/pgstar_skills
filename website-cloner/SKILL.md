# Website Cloner v2

自动化网站复刻工具 — 全量遍历、状态截图、动态录制、素材爬取、详细文档。

## 工作流程概览

```
输入URL → 分析结构 → 识别动态元素 → 全量遍历测试 → 状态截图+动态录制 → 爬取素材 → 生成文档
```

## 执行步骤

### 1. 启动与初始化

```
使用Playwright MCP：
1. 设置视口：1920x1080
2. 导航到用户提供的URL
3. 等待页面完全加载
4. 获取页面基础信息（标题、描述、OG标签）
```

### 2. 网站结构分析

```
分析并记录：
- 页面层级结构（导航、主要内容、侧边栏、页脚）
- 所有区域划分（hero、section、footer等）
- 所有可交互元素（按钮、链接、菜单、卡片、表单）
- 嵌入内容（图片、视频、音频、iframe）
- 动画触发元素（悬停效果、滚动动画、页面过渡）
```

### 3. 动态内容检测

```
检测每个元素是否为动态内容：
- CSS属性检测：animation、transition、@keyframes
- JS事件监听：animationstart、transitionend
- 帧对比：截图两帧，像素差异>5%判定为动态

标记元素类型：
- 静态元素：只需截图默认+悬停+点击状态
- 动态元素：需录制MP4/GIF
- 媒体元素：需下载原始资源
```

### 4. 素材资源爬取

```
使用全部方式识别素材：
- HTML标签：img、video、audio、source、picture
- CSS背景图：background-image、background
- 网络请求：拦截所有媒体资源请求

下载到assets/目录：
- 命名规则：{序号}-{位置}-{简短描述}.{扩展名}
- 示例：01-hero-bg.jpg、02-work-card-1.png
- 视频：下载原文件 + 截图封面帧
- 音频：下载原文件 + 截图播放器界面
```

### 5. 全量遍历测试

```
遍历所有区域的每个元素，执行以下操作：

对每个元素：
1. 滚动到元素位置
2. 截图默认状态（静止）
3. 悬停鼠标 → 截图悬停状态
4. 点击元素 → 截图点击后状态
5. 如果是动态元素 → 录制MP4/GIF

元素状态测试矩阵：
| 状态 | 操作 | 输出 |
|------|------|------|
| default | 无操作 | 静态截图 |
| hover | 鼠标悬停 | 静态截图 |
| active | 鼠标点击 | 静态截图 |
| dynamic | 触发动画 | MP4/GIF |
```

### 6. 动态效果录制

```
MP4录制（复杂动画）：
- 场景：页面加载动画、滚动视差、复杂CSS/JS动画
- 时长：1-3秒
- 格式：MP4

GIF录制（简单悬停）：
- 场景：按钮悬停、链接变色、简单状态切换
- 帧率：24fps
- 格式：GIF
```

## 调用方式

用户输入：
```
/website-cloner https://example.com
```

或：
```
帮我复刻这个网站：https://example.com
```

## 执行策略

采用**全量遍历策略**：
1. **第一阶段：结构分析** — 识别所有区域和元素
2. **第二阶段：动态检测** — 标记静态/动态元素
3. **第三阶段：素材爬取** — 下载所有媒体资源
4. **第四阶段：全量测试** — 遍历每个元素，测试三种状态
5. **第五阶段：动态录制** — 录制动画效果
6. **第六阶段：文档生成** — 整合所有信息

## 错误处理

遇到以下情况时**停止并报告**：
- URL无法访问
- 页面加载超时（30秒）
- Playwright MCP不可用
- 磁盘空间不足

## 注意事项

- 视口尺寸：1920x1080
- 不处理需要登录的内容
- 不执行破坏性操作（如下单、删除）
- 截图格式：PNG
- 动态录制：MP4（复杂）+ GIF（简单）
- 所有状态变化都会截图/录制
- 素材全部下载到assets目录
