# Website Cloner

自动化网站复刻工具 — 浏览、截图、测试、记录，生成完整复刻文档。

## 工作流程概览

```
输入URL → 分析网站结构 → 截图静态页面 → 测试交互 → 记录状态变化 → 生成文档
```

## 执行步骤

### 1. 启动与导航

```
使用Playwright MCP：
1. 打开新页面
2. 导航到用户提供的URL
3. 等待页面完全加载
4. 获取页面基础信息（标题、描述、OG标签）
```

### 2. 网站结构分析

```
分析并记录：
- 页面层级结构（导航、主要内容、侧边栏、页脚）
- 所有可点击元素（按钮、链接、菜单项）
- 表单元素（输入框、下拉框、复选框）
- 嵌入内容（图片、视频、iframe）
- 动画触发元素（悬停效果、滚动动画）
```

### 3. 静态截图

```
按顺序截图：
- 全页截图（完整页面）
- 首屏截图（视口大小）
- 各个区块截图（导航、主内容、页脚等）
命名：01-homepage-full.png, 01-homepage-hero.png, 01-homepage-nav.png
```

### 4. 交互测试

```
对每个可交互元素执行：
1. 悬停 → 截图状态变化
2. 点击 → 等待响应 → 截图结果
3. 填写表单 → 截图验证状态
4. 滚动 → 截图不同位置
5. 等待动画完成 → 截图最终状态
```

### 5. 状态记录

```
记录所有发现：
- 交互前后的视觉差异
- 动画类型和持续时间
- 颜色变化、位移、缩放等
- 响应式断点（如有）
```

## 目录结构

执行完成后在当前目录创建：

```
website-clone-{域名}/
├── README.md                    # 复刻文档（概览+附录）
├── screenshots/                 # 截图文件夹
│   ├── 01-homepage/
│   │   ├── 01-homepage-full.png
│   │   ├── 01-homepage-hero.png
│   │   ├── 01-homepage-nav.png
│   │   └── ...
│   ├── 02-about/
│   └── ...
└── assets/                      # 其他资源（如有）
```

## 截图命名规则

```
{序号}-{页面名}-{元素名}-{状态}.png

示例：
01-homepage-hero.png           # 首页英雄区
01-homepage-nav-hover.png      # 导航悬停状态
01-homepage-btn-click.png      # 按钮点击后
01-homepage-form-focus.png     # 表单聚焦状态
```

序号规则：按页面访问顺序递增（01, 02, 03...）

## 调用方式

用户输入：
```
/website-cloner https://example.com
```

或：
```
帮我复刻这个网站：https://example.com
```

## 执行策略

采用**混合策略**：
1. **第一阶段：快速概览** — 截图所有页面的静态状态
2. **第二阶段：深入测试** — 对关键页面进行完整交互测试
3. **第三阶段：文档生成** — 整合所有信息生成复刻文档

## 错误处理

遇到以下情况时**停止并报告**：
- URL无法访问
- 页面加载超时（30秒）
- Playwright MCP不可用
- 磁盘空间不足

## 注意事项

- 仅测试桌面端（视口1920x1080）
- 不处理需要登录的内容
- 不执行破坏性操作（如下单、删除）
- 截图质量自动调整，格式为PNG
- 所有状态变化都会截图记录
